# RAG (检索 增强 生成)

大模型的两个方向: RAG 和 Agent

## LLM 的幻觉问题

时间足够长猴子也能打出莎士比亚(llm是更大概率打出莎士比亚的猴子)

1. llm是从大量数据中训练出的`基于概率`来一个个生成token的模型;
2. llm本身没有逻辑,但是实际上模型已经足够大,也已经足够逻辑推理了
3. 这种涌现性智能的关键在于:训练数据做够大->模型概率才足够高->确实显得很智能

## LLM 领域知识薄弱

1. 训练数据更新慢(例如新闻)
2. 缺少行业资料(例如: 公司内部文档)

## RAG 原理

1. RAG 就是为了解决这两点问题(幻觉,领域知识薄弱)
2. RAG 本质上就是将 LLM 作为逻辑推理工具(减少幻觉),而不是信息检索工具(增加领域知识)

llm 是概率性的,那么就尽可能的提供正确的上下文信息.(减少幻觉) 就是给猴子身边放一本莎士比亚的书.

RAG基本流程

用户提出问题

1. 检索: 从向量数据库中检索与问题最相关的文档
2. 增强: 将检索到的文档作为上下文,与用户问题一起输入到 LLM 中; 例如:"仅依赖下述文档内容,回答用户问题"
3. 生成: LLM 生成回答

## RAG chat bot 实现流程

### 1. 建立向量数据库

1. 加载数据源 (pdf,code,数据库) 可利用langchain提供的工具
2. 切分数据 模型上下文窗口有限(例如 128k), 对大文档需要语义化的拆分为多个切片;
3. 嵌入(Embedding) 每个切片转换为向量表示的数据库中(vector store); 可以根据向量计算相似度,找到最相关的文档

### 2. 检索

有了向量数据库,就可以根据用户问题,从数据库中检索最相关的文档.

这里,用什么样的算法计算向量之间的距离,也是需要考虑的问题.

### 3. 增强 prompt

在检索到相关文档后, 就可以构建提示(prompt)了. 一般格式类似:

```
你是一个 xxx 的聊天机器人，你的任务是根据给定的文档回答用户问题，并且回答时仅根据给定的文档，尽可能回答用户问题。如果你不知道，你可以回答“我不知道”。

这是文档:
{docs}

用户的提问是:
{question}
```

### 4. 生成

最后就是将组织好的prompt,输入到 LLM 中,利用LLM的逻辑推理能力生成回答.
